Sammanfattning av projekt och reflektion kring styrkor och ev brister

Jag har nu uppdaterat min webbsida från det senaste projektet med både JavaScript och JSON-element. Jag har lagt till flera dynamiska JavaScript-funktioner, kopplat samman min “about-page” med JSON-data och hämtar mina projekt från min Github via REST API. De nya dynamiska elementen består främst av show/hide-funktioner och olika visningseffekter, som jag har implementerat både på projektsidan och about-sidan. På projektsidan lät jag min kreativitet flöda och skapade en scroll-effekt där projekten flyger in från sidan med hjälp av animationer när man skrollar.

Det var inte särskilt svårt att skapa effekten initialt, men när jag senare kopplade sidan till GitHub via REST API märkte jag att alla animationer och show/hide-funktioner slutade fungera. Detta ledde till viss frustration, och jag insåg snabbt att jag behövde bygga om och förbättra min JavaScript-kod. Efter mycket trial and error fick jag slutligen allt att fungera som jag hade tänkt, vilket jag nu är väldigt stolt över.

Jag tror att styrkan i mitt projekt ligger i moderniteten och enkelheten jag lyckats skapa. Min portfolio är lättnavigerad och stilren vilket gör det enkelt för besökaren att hitta information utan att störas av onödigt krångel. Mycket tid har även lagts ner för att göra webbsidan tydlig och enhetlig i form av “hover-states” och subtila animationer vilket jag tycker jag lyckats med. De dynamiska JavaScript-elementen, som show/hide-funktioner och scroll-animationer bidrar till en mer engagerande användarupplevelse och ger sidan liv, samtidigt som JSON-integrationen gör den flexibel och lätt att uppdatera med nya projekt. Sammantaget tycker jag att sidan erbjuder en bra balans mellan estetik och funktionalitet vilket jag anser är centralt för en bra UX-design.

Där sidan däremot brister är i nytänkande och unik identitet. Även om designen är stilren och modern, upplever jag att den riskerar att smälta in bland många andra portfolior med liknande upplägg. För att sidan ska sticka ut mer skulle jag vilja experimentera med mer originella designval som exempelvis ovanliga layoutlösningar, interaktiva element eller visuella teman som stärker min personliga stil. På så sätt skulle sidan inte bara vara funktionell och snygg, utan även lämna ett starkare intryck hos besökaren.

Frågor

Vad kan man utveckla med hjälp av Javascript inom frontend?

Med JavaScript inom frontend kan man göra en webbsida dynamisk och interaktiv genom funktionella element som reagerar på användarens handlingar. Detta gör det möjligt att skapa mer engagerande, roliga och komplexa webbsidor istället för att webben bara skulle vara statisk och ”platt”. JavaScript gör det möjligt att bestämma när och hur element ska visas, beroende på användarens klick, scrollning eller andra interaktioner. Det introducerar ett slags logiskt och matematiskt tänk till webben, där utvecklaren kan kombinera olika funktioner och regler för att skapa kreativa lösningar. Resultatet är nästan obegränsade möjligheter, allt från dynamiska animationer till mer avancerade applikationer vilket gör webben både mer levande och spännande.

Vad är JSON och hur används det inom frontend?

JSON står för JavaScript Object Notation och är en form av datalagring som är inbyggd i JavaScript. Det gör det möjligt att enkelt lagra och hantera data på en webbplats, vilket underlättar utvecklingen av större och mer komplexa applikationer. JSON representerar data i nyckel–värde-par, vilket gör den både lättläst för människor och lätt att bearbeta för datorer. Inom frontend används JSON ofta för att hämta data från servern, skicka data till backend och att lagra information lokalt i webbläsaren. Detta till exempel med “localStorage” eller “sessionStorage”. Tack vare detta kan webbsidor uppdateras dynamiskt och hantera information på ett strukturerat sätt vilket är avgörande för moderna webbapplikationer.

Vad är HTTP och varför bör man som frontendutvecklare ha kunskap om det och dess protokoll?

HTTP står för HyperText Transfer Protocol och är protokollet som styr hur data skickas och tas emot mellan webbläsare och servrar på webben. När en användare besöker en webbsida skickar webbläsaren en HTTP-förfrågan (request) till servern, som svarar med en HTTP-respons, ofta innehållande HTML, CSS, JavaScript eller JSON-data. HTTP använder olika metoder, som “GET” för att hämta data, “POST” för att skicka data, samt “PUT”, “PATCH” och “DELETE” för att uppdatera eller ta bort information. För frontendutvecklare är det viktigt att förstå HTTP eftersom det gör det möjligt att följa dataflödet mellan klient och server, felsöka problem med statuskoder som 404 eller 500, hantera säkerhet och prestanda samt interagera effektivt med API:er som REST eller GraphQL. Kunskap om HTTP ger dessutom insikt i hur cookies, CORS och autentisering fungerar vilket är avgörande för att bygga moderna, dynamiska och säkra webbapplikationer.
